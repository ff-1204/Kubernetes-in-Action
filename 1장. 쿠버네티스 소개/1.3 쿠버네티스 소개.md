# 1.3 쿠버네티스 소개

수많은 컨테이너를 사람이 일일이 관리하는 것은 불가능합니다. 쿠버네티스는 이 과정을 자동화하여 시스템이 항상 사용자가 설정한 **'원하는 상태(Desired State)'** 를 스스로 유지하도록 만드는 지능형 자동화 도구입니다.

### 쿠버네티스의 기원
구글이 수만 대의 서버를 관리하며 쌓은 '보그(Borg)' 시스템의 노하우를 바탕으로 탄생했습니다. 세계 최대 규모의 서비스를 운영해 온 **구글의 검증된 기술력을 오픈소스로 공개한 결과물**입니다.

### 인프라 추상화와 가치 중심 개발
쿠버네티스는 복잡한 서버 환경을 하나로 묶어 관리해 줍니다.
* **인프라 추상화**: 개발자는 개별 서버의 위치나 구체적인 사양을 몰라도 됩니다. 클러스터 전체를 하나의 거대한 자원으로 보고 애플리케이션만 배포하면 됩니다.
* **비즈니스 본질 집중**: 서버 설정이나 네트워크 구성 같은 관리 부담에서 벗어나, 서비스의 핵심 기능 개선과 **애플리케이션의 본질적인 가치** 를 높이는 일에만 전념할 수 있습니다.

### 선언적 모델 (Declarative Model)
쿠버네티스는 결과만 정의하면 과정은 알아서 처리하는 **'선언적 모델'** 방식을 따릅니다.
* **개념**: 컴퓨터에게 절차를 일일이 지시하는 **명령형(Imperative)** 과 달리, **"무엇이 되어야 하는지(What)"** 라는 최종 결과만 알려주면 시스템이 알아서 그 상태로 만들어 주는 방식입니다.
* **방식**: "어디에 앱을 띄워라"고 일일이 명령하는 대신, "이 앱의 복제본이 항상 3개 유지되어야 해"라고 **최종 목표 상태만 선언**합니다.
* **자동 유지보수**: 만약 장애로 인해 앱이 중단되더라도, 쿠버네티스가 이를 즉각 감지하고 새 앱을 띄워 약속된 상태를 스스로 복구합니다.
* **YAML로 정의**: 쿠버네티스에서는 이 '원하는 상태'를 사람이 읽기 쉬운 **YAML 파일(매니페스트)** 에 작성해 클러스터에 제출하면, 쿠버네티스가 그 상태를 자동으로 맞춰 줍니다.

### 쿠버네티스의 핵심 장점 3가지
1. **자동 복구 (Self-healing)**: 고장 난 컨테이너를 스스로 감지하고 건강한 컨테이너로 즉시 교체합니다.
2. **수평 확장 (Scaling)**: 트래픽이 몰리는 상황을 감지하여 자동으로 애플리케이션 개수를 늘려 대응합니다.
3. **효율적 자원 사용 (Bin Packing)**: 서버의 여유 공간을 분석하여 컨테이너를 가장 적절한 곳에 배치함으로써 서버 자원의 낭비를 최소화합니다.

## 결론: 쿠버네티스의 역할

* **운영 지옥의 해방구**: 수동 관리의 한계를 넘어 **수천 개의 서비스**를 사람 대신 **24시간 감시하고 조율**하는 자동화 체계를 구축합니다.
* **목표 기반 자동화**: 사용자가 설정한 목표 상태를 시스템이 스스로 인지하고 유지함으로써, 장애 복구와 자원 할당을 효과적으로 자동화합니다.
