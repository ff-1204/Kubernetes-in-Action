# 2.1 쿠버네티스 클러스터 설치

본격적인 학습 전에 **나만의 작은 쿠버네티스 클러스터(실습 환경)**를 준비하는 단계입니다. 인프라는 책으로만 읽어서는 절대 익혀지지 않으므로, 이 절에서는 **실습 환경을 갖추고 `kubectl`이라는 입을 통해 클러스터와 대화할 수 있도록** 만드는 데 집중합니다.

---

## 클러스터 설치는 부록 A 참고
실제 설치 절차(WSL2 + Docker Engine + kind + kubectl)는 환경별로 분기가 많고 분량이 길어, 본문이 아닌 별도 문서로 분리해 두었습니다.

* **설치 가이드 위치:** [부록 A. 쿠버네티스 실습 환경 구축](../부록%20A.%20쿠버네티스%20실습%20환경%20구축)

---

## kubectl: 클러스터와 대화하는 입
`kubectl`은 쿠버네티스 API 서버에 명령을 보내는 **공식 CLI 도구**입니다. 책 전반에 걸쳐 가장 많이 입력하게 될 한 단어입니다.

* **조회와 진단:** `get`, `describe`, `logs`, `exec`로 클러스터의 현재 상태를 들여다봄
* **선언적 명령:** YAML로 원하는 상태를 정의하고 `apply`로 적용
* **명령형 명령:** `run`, `create`, `expose`, `scale` 같이 즉시 결과를 만드는 한 줄 명령


---

## 클러스터 정상 동작 확인

설치가 끝났다면 다음 세 단계로 클러스터의 생사부터 점검합니다.

### 1) 클러스터 작동 여부

API 서버(컨트롤 플레인)와 클러스터 내부 DNS의 엔드포인트가 출력되면 클러스터가 정상 작동하고 있다는 뜻입니다.

```bash
ysmoon@YSMOON:~$ kubectl cluster-info
Kubernetes control plane is running at https://127.0.0.1:xxxxx
CoreDNS is running at https://127.0.0.1:xxxxx/api/v1/namespaces/kube-system/services/kube-dns:dns/proxy

To further debug and diagnose cluster problems, use 'kubectl cluster-info dump'.
```

### 2) 노드 상태

클러스터에 등록된 노드(서버) 목록과 상태를 출력합니다. control-plane 노드가 `Ready` 상태로 나오면 정상입니다.

```bash
ysmoon@YSMOON:~$ kubectl get nodes
NAME                   STATUS   ROLES           AGE     VERSION
ysmoon-control-plane   Ready    control-plane   5m59s   v1.35.0
```

### 3) 시스템 파드

`-A`(또는 `--all-namespaces`) 옵션을 쓰면 모든 네임스페이스의 파드를 한 번에 볼 수 있습니다. 컨트롤 플레인 파드는 `kube-system`에 몰려 있으므로, 한눈에 모두 `Running` 상태인지 확인할 수 있습니다.

```bash
ysmoon@YSMOON:~$ kubectl get pods -A
NAMESPACE            NAME                                           READY   STATUS    RESTARTS   AGE
kube-system          coredns-7d764666f9-stwtk                       1/1     Running   0          8m5s
kube-system          coredns-7d764666f9-ts57x                       1/1     Running   0          8m5s
kube-system          etcd-ysmoon-control-plane                      1/1     Running   0          8m10s
kube-system          kindnet-wspvh                                  1/1     Running   0          8m5s
kube-system          kube-apiserver-ysmoon-control-plane            1/1     Running   0          8m10s
kube-system          kube-controller-manager-ysmoon-control-plane   1/1     Running   0          8m10s
kube-system          kube-proxy-xhwwb                               1/1     Running   0          8m5s
kube-system          kube-scheduler-ysmoon-control-plane            1/1     Running   0          8m12s
local-path-storage   local-path-provisioner-67b8995b4b-bz8q4        1/1     Running   0          8m5s
```

표준 K8s 컨트롤 플레인 컴포넌트(`etcd`, `kube-apiserver`, `kube-controller-manager`, `kube-scheduler`)가 모두 `Running` 상태로 보이면 성공입니다.

## 효율을 끌어올리는 별칭(alias)과 자동완성 설정

### 1) 별칭(alias) 설정: 7글자를 1글자로
```bash
# 별칭 설정
ysmoon@YSMOON:/mnt/c/windows/system32$ echo 'alias k=kubectl' >> ~/.bashrc
ysmoon@YSMOON:/mnt/c/windows/system32$ source ~/.bashrc
# 설정 확인
ysmoon@YSMOON:/mnt/c/windows/system32$ k get nodes
NAME                   STATUS   ROLES           AGE     VERSION
ysmoon-control-plane   Ready    control-plane   2d21h   v1.35.0
```
* 매번 `kubectl` 대신 `k` 한 글자로 명령을 보낼 수 있습니다.

### 2) 명령줄 자동완성 (Tab Completion)
하위 명령과 리소스 이름을 **Tab 키로 자동완성**하게 만들어 주는 설정입니다.

> 💡 아래 코드 블록의 `<TAB>`은 **Tab 키를 한 번 누른다는 표시**입니다(실제로 입력하는 글자가 아닙니다).

```bash
# 자동완성 설정
ysmoon@YSMOON:/mnt/c/windows/system32$ echo 'source <(kubectl completion bash)' >> ~/.bashrc
ysmoon@YSMOON:/mnt/c/windows/system32$ echo 'complete -F __start_kubectl k' >> ~/.bashrc
ysmoon@YSMOON:/mnt/c/windows/system32$ source ~/.bashrc

# 설정 확인 1-1: `k get no` 까지만 입력 후 Tab 키
ysmoon@YSMOON:/mnt/c/windows/system32$ k get no<TAB>
# 설정 확인 1-2: 나머지 글자(`des`)가 자동으로 채워집니다.
ysmoon@YSMOON:/mnt/c/windows/system32$ k get nodes

# 설정 확인 2-1: `k logs -n kube-system kube-a` 까지만 입력 후 Tab 키
ysmoon@YSMOON:/mnt/c/windows/system32$ k logs -n kube-system kube-a<TAB>
# 설정 확인 2-2: 나머지 글자(`piserver-ysmoon-control-plane`)가 자동으로 채워집니다.
ysmoon@YSMOON:/mnt/c/windows/system32$ k logs -n kube-system kube-apiserver-ysmoon-control-plane
```
* Pod 이름이나 네임스페이스를 외워 칠 필요 없이 **Tab 한 번에 끝까지 채워주는** 강력한 기능입니다.
* 오타로 인한 실수가 줄고 디버깅 속도가 눈에 띄게 빨라집니다.

## 요약 및 비유

| 개념 | 비유 | 기술적 의미 |
| :--- | :--- | :--- |
| **클러스터** | **나만의 작은 마라탕 가게** | 학습용 단일 노드 쿠버네티스 환경 |
| **kubectl** | **주방에 주문을 넣는 키오스크** | API 서버와 대화하는 공식 CLI |
| **alias / 자동완성** | **단축 버튼과 메뉴 추천** | 반복 입력 비용을 줄이는 보조 설정 |

---

## 결론: 출발선을 짧게, 본문은 길게
* 클러스터 설치는 **목적이 아닌 출발선**입니다. 부록 A로 환경을 단번에 끝내고, 이후에는 **그 위에서 어떤 일을 시킬 수 있는가**에 시간을 투자하세요.
* alias 한 줄과 자동완성 한 줄이 사소해 보이지만, 책을 끝낼 때쯤이면 **수천 번의 명령어 입력 비용**을 절약해 줍니다.
